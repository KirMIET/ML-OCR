# OCR Model Training and Inference Guide

Данный документ описывает процесс обучения моделей и запуска инференса для трёх вариантов моделей PARSeq.

## Структура проекта

- **parseq_swa/** - Модель с Stochastic Weight Averaging (SWA) на основном датасете
- **parseq_sint/** - Модель с синтетическими данными (SINT)
- **parseq_sint_crop/** - Модель с синтетическими данными и кропированием (SINT + CROP)

## Модель 2: parseq_sint (Синтетические данные)

### Обучение

1. **Подготовка синтетических данных:**
   ```bash
   cd parseq_sint
   python sint_data.py
   ```

   Этот скрипт создаёт синтетические изображения в `./dataset/sint_data/`.

2. **Подготовка объединённого датасета:**
   ```bash
   python all_data.py
   ```

   Этот скрипт:
   - Объединяет реальные изображения из `./dataset/train/train/` (префикс: `real_`)
   - Добавляет синтетические данные из `./dataset/sint_data/` (префикс: `sint_`)
   - Создаёт объединённый датасет в `./dataset/full_data/`
   - Создаёт файл меток `./dataset/full_labels.csv`

3. **Запуск обучения:**
   ```bash
   python train_parseq_sint.py
   ```

   **Параметры обучения:**
   - Датасет: `./dataset/full_labels.csv` + `./dataset/full_data/`
   - Эпохи: 20 (с SWA с 15-й эпохи)
   - Batch Size: 64
   - Learning Rate: 7e-4 (базовая), 5e-5 (SWA)
   - Размер изображения: 192x64
   - Словарь: "0123456789"

3. **Результат:**
   - Модель сохраняется в `./outputs/saved_models/parseq_sint.pth`

### Инференс

```bash
python ../inference_v5.py
```

Модель автоматически загружается и используется в ансамбле с весом **0.9**.

## Модель 3: parseq_sint_crop (Синтетические данные + кропирование)

### Обучение

1. **Создание синтетических данных:**
   ```bash
   cd parseq_sint_crop
   python sint_data.py
   ```

   Этот скрипт создаёт синтетические изображения в `./dataset/sint_data/`.

2. **Первое объединение данных (реальные + синтетические):**
   ```bash
   python all_data.py
   ```

   Объединяет реальные изображения из `./dataset/train/train/` и синтетические из `./dataset/sint_data/`:
   - Создаёт `./dataset/full_data/`
   - Создаёт `./dataset/full_labels.csv`

3. **Кропирование изображений:**
   ```bash
   python src_crop_number.py
   ```

   Этот скрипт:
   - Обрабатывает изображения из `./dataset/full_data/`
   - Применяет кропирование по границам текста
   - Сохраняет кропированные изображения в новой директории

4. **Подготовка нового датасета:**
   ```bash
   python src_new_dataset.py
   ```

   Этот вспомогательный скрипт подготавливает дополнительные кропированные данные.

5. **Второе объединение данных (с кропированными изображениями):**
   ```bash
   python all_data.py
   ```

   **Важно:** перед запуском необходимо изменить пути в скрипте `all_data.py` на пути к кропированным датасетам:
   - Измените `REAL_IMG_DIR` на путь к кропированным реальным изображениям
   - Измените `SINT_IMG_DIR` на путь к кропированным синтетическим изображениям
   - Убедитесь, что `OUTPUT_IMG_DIR` и `OUTPUT_CSV_PATH` указывают на новые директории

   Повторно объединяет данные с учётом кропированных изображений.

6. **Запуск обучения:**
   ```bash
   python train_parseq.py
   ```

   **Параметры обучения:**
   - Датасет: `./dataset/full_labels.csv` + кропированные изображения
   - Эпохи: 30 (с SWA с 22-й эпохи)
   - Batch Size: 64
   - Learning Rate: 7e-4 (базовая), 5e-5 (SWA)
   - Weight Decay: 0.05
   - Gradient Clip: 5.0
   - Размер изображения: 192x64
   - Словарь: "0123456789"

4. **Результат:**
   - Модель сохраняется в `./outputs/saved_models/parseq_sint_crop.pth`

### Инференс

```bash
python ../inference_v5.py
```

Модель автоматически загружается и используется в ансамбле с весом **1.25** (наибольший вес).

## Запуск ансамблевого инференса

Все три модели используются вместе в ансамбле:

```bash
python inference_v5.py
```

**Конфигурация ансамбля:**
- `parseq_swa.pth` → вес: 1.0
- `parseq_sint.pth` → вес: 0.9
- `parseq_sint_crop.pth` → вес: 1.25

**Параметры инференса:**
- Источник: `./dataset/test/test/`
- Выход: `./submissions/ensemble_weighted_and_gamma_v5.csv`
- Gamma коррекция: включена (GAMMA_VALUE = 5.0)
- Auto crop: отключен
- TTA (Test Time Augmentation): используется с несколькими вариантами изображения
- Refinement steps: 7

**Особенности инференса:**
- Для каждого изображения генерируются варианты (CLAHE, Gamma, Резкость)
- Модели обрабатывают все варианты
- Результаты усредняются с учётом весов моделей

## Важные замечания

1. **Порядок выполнения скриптов для parseq_sint_crop:**
   - `sint_data.py` → создание синтетических данных
   - `all_data.py` (1-й раз) → объединение реальных и синтетических
   - `src_crop_number.py` → кропирование объединённых изображений
   - `src_new_dataset.py` → подготовка датасета
   - `all_data.py` (2-й раз) → повторное объединение кропированных данных

2. **Изменение путей в all_data.py:**
   Перед вторым запуском `all_data.py` в parseq_sint_crop необходимо изменить пути в скрипте:
   - `REAL_IMG_DIR` - установить путь к кропированным реальным изображениям
   - `SINT_IMG_DIR` - установить путь к кропированным синтетическим изображениям
   - `OUTPUT_IMG_DIR` - новая директория для объединённых кропированных данных
   - `OUTPUT_CSV_PATH` - новый CSV файл для меток кропированных данных

3. **Требования к датасету**:
   - `dataset/train.csv` - файл с метками реальных изображений (колонки: Filename, Price)
   - `dataset/train/train/` - директория с реальными изображениями
   - `dataset/sint_data.csv` - файл с метками синтетических данных
   - `dataset/sint_data/` - директория с синтетическими изображениями
   - `dataset/test/test/` - тестовые изображения для инференса

3. **Зависимости:**
   - torch, torchvision
   - opencv-python
   - albumentations
   - pandas, numpy
   - scikit-learn
   - tqdm
   - editdistance

4. **GPU рекомендуется** для обучения. Скрипты автоматически используют CUDA если доступно.
